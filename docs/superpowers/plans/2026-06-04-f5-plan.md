# F5 — Pi-first multi-host parity + installer matrix Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Ship opsbench as a Pi-first multi-host toolkit with Claude Code / Codex CLI / GitHub Copilot / Cursor / Gemini / OpenCode parity adapters and distro-packaged installers (Homebrew tap, AUR PKGBUILD, Nix flake).

**Architecture:** Pi becomes the canonical authoring host. A generator (`tools/pi-compat-layer/adapt.sh`) reads opsbench `SKILL.md` / `AGENT.md` files and emits Pi-native manifests plus AGENTS.md fragments, models.json registrations, and pre/post-exec hooks. The five other host adapters (`codex/copilot/cursor/gemini/opencode-compat-layer/`) are projections from the Pi output through a shared library (`tools/_lib/adapt-common.sh`). The installer matrix wraps the whole tree as Homebrew, AUR, and Nix packages, all defaulting to Pi-first layout with explicit overrides for legacy hosts.

**Tech Stack:** Bash (adapters, hooks, install scripts), JSON Schema (manifest validation), YAML (CI + marketplace manifest), Ruby (Homebrew formula), PKGBUILD (Arch), Nix (flake), GitHub Actions (CI matrix), `jq` (manifest merging), `npx ajv-cli` (schema validation), `shellcheck` (lint).

**Parent spec:** [`../specs/2026-06-04-f5-design.md`](../specs/2026-06-04-f5-design.md)
**Parent roadmap:** [`../specs/2026-06-04-f-series-master-roadmap.md`](../specs/2026-06-04-f-series-master-roadmap.md)

## Preflight gating

- **PF.1.** F0 (recipes) merged. `git log origin/main | grep "F0 bulk-ship"` succeeds. If not, F5 is blocked — Pi adapter relies on the recipe + skill tree shape that F0 establishes.
- **PF.2.** F1 (Cedar-for-agents) merged. `tools-generated.cedar` exists in main. If not, the Pi pre-exec hook has no allowlist to enforce.
- **PF.3.** F3 (signed receipts) merged OR explicitly deferred. If F3 has merged, `scripts/verify-receipts.sh` is shipped inside the installer payload; if not, this is documented in `tools/installer/README.md` as a known gap with a follow-up tracking issue.

F2 (`opsbench-gateway`) is NOT a hard preflight — F5 ships gateway URL examples in Pi docs but the gateway itself is optional from F5's perspective.
