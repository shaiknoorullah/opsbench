# F1 — Cedar-for-agents adoption Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Steps use checkbox (`- [ ]`) syntax. Each step is sized to 2-5 min of agent work; the largest are the recipe-backfill loops where the agent fans out across files.

**Goal:** Replace hand-written per-agent Cedar allowlists with a build-time generator that reads each MCP recipe's `tools:` frontmatter block, derives least-privilege Cedar rules per agent class via `cedar-policy/cedar-for-agents`, and emits a gitignored `tools-generated.cedar` alongside the existing hand-written override; ship a Pi adapter so Pi-first hosts get a usable allowlist manifest in the same phase.

**Architecture:** Pure build-time tooling. A bash generator (`scripts/generate-cedar-policy.sh`) walks `packages/team-incident-response/mcp-recipes/*.md`, extracts the `tools:` YAML block via `yq`, joins it with a Cedar schema (`opsbench-schema.cedarschema.json`) and a Handlebars-style template (`agent-class.cedar.tmpl`), then shells to the upstream `cedar-for-agents` CLI to emit Cedar policy. A Pi adapter under `tools/pi-compat-layer/cedar-emit.sh` projects the same Cedar onto a Pi tool-allowlist JSON. A new CI workflow (`validate-cedar.yml`) runs the generator on every PR, validates with the Cedar CLI, snapshot-diffs against the last main run, and posts a sticky PR comment.

**Tech Stack:** Bash 4+, `yq` (mikefarah v4), `jq`, `cedar-policy-cli` (cargo), `cedar-for-agents` (cargo, SHA-pinned), GitHub Actions, lefthook (existing), Pi `AGENTS.md`/`SYSTEM.md` convention.

**Parent spec:** [`../specs/2026-06-04-f1-design.md`](../specs/2026-06-04-f1-design.md)

- [ ] **1.3** Write `scripts/install-cedar-tools.sh`. The script:
  - Detects OS (`uname -s`) and arch (`uname -m`).
  - On macOS: prefers `brew install cedar-policy/tap/cedar` if a tap exists, falls back to `cargo install`.
  - On Linux: uses `cargo install` (assumes `rustup` is present; emits a clear error otherwise).
  - Pins `cedar-policy-cli` to a known-good tag (e.g. `--version 4.4.0` — confirm against crates.io on the day).
  - Pins `cedar-for-agents` to a git SHA (e.g. `--git https://github.com/cedar-policy/cedar-for-agents --rev <SHA>`); the SHA is recorded in a comment header and in `scripts/test-cedar-tools.sh`.
  - Is idempotent: checks `command -v cedar` and `command -v cedar-for-agents` before installing.
  - Exits 0 on success and prints the installed version of each binary to stdout.

  ```bash
  #!/usr/bin/env bash
  # F1 — Install Cedar tooling (cedar CLI + cedar-for-agents generator).
  set -euo pipefail
  CEDAR_CLI_VERSION="4.4.0"                 # crates.io: cedar-policy-cli
  CEDAR_FOR_AGENTS_REV="REPLACE_WITH_SHA"   # pinned during implementation; recorded here
  CEDAR_FOR_AGENTS_REPO="https://github.com/cedar-policy/cedar-for-agents"

  log() { printf '[install-cedar-tools] %s\n' "$*"; }

  ensure_rust() {
    command -v cargo >/dev/null && return 0
    log "ERR: cargo missing. Install rustup from https://rustup.rs/ then re-run." >&2
    exit 2
  }

  install_cedar_cli() {
    if command -v cedar >/dev/null && cedar --version | grep -q "$CEDAR_CLI_VERSION"; then
      log "cedar CLI $CEDAR_CLI_VERSION already installed; skipping."
      return 0
    fi
    log "Installing cedar-policy-cli@$CEDAR_CLI_VERSION via cargo..."
    cargo install cedar-policy-cli --version "$CEDAR_CLI_VERSION" --locked
  }

  install_cedar_for_agents() {
    if command -v cedar-for-agents >/dev/null; then
      log "cedar-for-agents already installed; skipping (rev pinning enforced in CI)."
      return 0
    fi
    log "Installing cedar-for-agents from $CEDAR_FOR_AGENTS_REPO @ $CEDAR_FOR_AGENTS_REV..."
    cargo install --git "$CEDAR_FOR_AGENTS_REPO" --rev "$CEDAR_FOR_AGENTS_REV" --locked cedar-for-agents
  }

  ensure_rust
  install_cedar_cli
  install_cedar_for_agents

  log "Versions:"
  cedar --version
  cedar-for-agents --version || log "WARN: cedar-for-agents --version not implemented upstream; smoke test will exercise"
  ```

## Task 4 — Generator script (scripts/generate-cedar-policy.sh)

**Files:** Create — scripts/generate-cedar-policy.sh.

- [ ] **4.1** Author the generator following spec §3.2. Implementation notes:
  - Use `set -euo pipefail`. Trap `EXIT` to clean `WORK_DIR`.
  - Required tools: `yq`, `jq`, `cedar`, `cedar-for-agents`. Each missing tool exits with a precise error pointing at `scripts/install-cedar-tools.sh`.
  - Walk every `.md` file under `RECIPES_DIR` unless `--only PATH` is passed (used by the smoke test).
  - Skip files whose basename starts with `README` or `_` (private fixtures).
  - Extract frontmatter with the awk trick (lines between the first two `---` markers).
  - Convert to JSON via `yq -o=json '. | {recipe_id, upstream, tools}'`. If `tools` is missing or null, log `skip (no tools): $recipe` to stderr and continue.
  - Cross-check `suggested_agent_classes` references against the agent directory listing (use `find packages/team-incident-response/agents -name '*.md'` to build the allowed set; strip `.md`); unknown classes are WARN (today) and will become hard-fail in F4.
  - Merge all per-recipe blocks into one JSON array.
  - Invoke `cedar-for-agents convert --input <merged-tools.json> --schema <schema> --template <template> --emit-cedar > <output>.new`.
  - Run `cedar validate --schema <schema> --policies <output>.new`. On failure: keep the `.new` file for debugging; exit non-zero.
  - Atomic install: `mv <output>.new <output>`.
  - Print a summary line: `[ok] wrote <output> (recipes: N, permit-rules: M)`.
  - Accept flags `--output PATH` and `--only PATH` (both optional; defaults are `tools-generated.cedar` and "walk all recipes").
- [ ] **4.2** Generator skeleton (see spec §3.2 for the long form; this is the executable variant):
  ```bash
  #!/usr/bin/env bash
  # F1 — Cedar policy generator. Reads recipe frontmatter, emits tools-generated.cedar.
  set -euo pipefail

  OPSBENCH_ROOT="$(cd "$(dirname "$0")/.." && pwd)"
  RECIPES_DIR="${OPSBENCH_ROOT}/packages/team-incident-response/mcp-recipes"
  POLICIES_DIR="${OPSBENCH_ROOT}/packages/team-incident-response/policies/cedar"
  SCHEMA_PATH="${POLICIES_DIR}/opsbench-schema.cedarschema.json"
  TEMPLATE_PATH="${POLICIES_DIR}/templates/agent-class.cedar.tmpl"
  DEFAULT_OUTPUT="${POLICIES_DIR}/tools-generated.cedar"
  ONLY_PATH=""
  OUTPUT_PATH="$DEFAULT_OUTPUT"

  # Parse flags.
  while [[ $# -gt 0 ]]; do
    case "$1" in
      --output) OUTPUT_PATH="$2"; shift 2 ;;
      --only)   ONLY_PATH="$2"; shift 2 ;;
      -h|--help) sed -n '2,16p' "$0"; exit 0 ;;
      *) echo "unknown flag: $1" >&2; exit 2 ;;
    esac
  done

  WORK_DIR="$(mktemp -d)"
  trap 'rm -rf "$WORK_DIR"' EXIT

  for tool in yq jq cedar cedar-for-agents; do
    command -v "$tool" >/dev/null || {
      echo "ERR: $tool not installed; run scripts/install-cedar-tools.sh" >&2; exit 2;
    }
  done

  # 1) Build the recipe set.
  if [[ -n "$ONLY_PATH" ]]; then
    RECIPE_FILES=("$ONLY_PATH")
  else
    mapfile -t RECIPE_FILES < <(find "$RECIPES_DIR" -maxdepth 1 -name '*.md' \
      ! -name 'README*' ! -name '_*' | sort)
  fi

  # 2) Walk recipes, extract tools blocks.
  TOOLS_JSON="${WORK_DIR}/tools.json"
  echo "[]" > "$TOOLS_JSON"
  for recipe in "${RECIPE_FILES[@]}"; do
    frontmatter="$(awk '/^---$/{c++; next} c==1{print} c>=2{exit}' "$recipe")"
    [[ -z "$frontmatter" ]] && { echo "skip (no frontmatter): $recipe" >&2; continue; }
    block="$(printf '%s\n' "$frontmatter" | yq -o=json '. | {recipe_id, upstream, tools}')"
    if ! echo "$block" | jq -e '.tools // empty | length > 0' >/dev/null 2>&1; then
      echo "skip (no tools): $recipe" >&2
      continue
    fi
    tmp="${WORK_DIR}/_tmp.json"
    jq --argjson r "$block" '. + [$r]' "$TOOLS_JSON" > "$tmp" && mv "$tmp" "$TOOLS_JSON"
  done

  # 3) Cross-check agent classes.
  AGENT_LIST="${WORK_DIR}/agents.txt"
  find "${OPSBENCH_ROOT}/packages/team-incident-response/agents" -name '*.md' \
    | xargs -n1 basename | sed 's/\.md$//' | sort -u > "$AGENT_LIST"
  echo "human-operator" >> "$AGENT_LIST"
  UNKNOWN="$(jq -r '[.[].tools[].suggested_agent_classes[]] | unique | .[]' "$TOOLS_JSON" \
    | sort -u | grep -vxFf "$AGENT_LIST" || true)"
  if [[ -n "$UNKNOWN" ]]; then
    echo "WARN: unknown agent classes referenced:" >&2
    printf '  - %s\n' $UNKNOWN >&2
  fi

  # 4) Emit Cedar.
  cedar-for-agents convert \
    --input "$TOOLS_JSON" \
    --schema "$SCHEMA_PATH" \
    --template "$TEMPLATE_PATH" \
    --emit-cedar \
    --sort-by action,principal \
    > "$OUTPUT_PATH.new"

  # 5) Validate.
  cedar validate --schema "$SCHEMA_PATH" --policies "$OUTPUT_PATH.new"

  # 6) Atomic install + summary.
  mv "$OUTPUT_PATH.new" "$OUTPUT_PATH"
  RECIPE_COUNT="$(jq 'length' "$TOOLS_JSON")"
  PERMIT_COUNT="$(grep -c '^permit' "$OUTPUT_PATH" || echo 0)"
  echo "[ok] wrote $OUTPUT_PATH (recipes: $RECIPE_COUNT, permit-rules: $PERMIT_COUNT)"
  ```

  #!/usr/bin/env bash
  set -euo pipefail
  ROOT="$(cd "$(dirname "$0")/../.." && pwd)"
  CEDAR="$ROOT/packages/team-incident-response/policies/cedar/tools-generated.cedar"
  OVERLAY="$ROOT/tools/pi-compat-layer/pi-overlay.yaml"

  python3 - "$CEDAR" "$WRITE_OWNER" "$OUT" <<'PY'
  import json, re, sys, collections, datetime
  cedar_path, write_owner, out = sys.argv[1], sys.argv[2], sys.argv[3]
  src = open(cedar_path).read()
  blocks = re.split(r'(?m)^permit\s*\(', src)[1:]
  agents = collections.defaultdict(lambda: {"allow_tools": set(), "deny_tools": set(), "notes": ""})
  for b in blocks:
      principals = re.findall(r'User::"([^"]+)"', b)
      action = re.search(r'Action::"([^"]+)"', b).group(1)
      guarded = "human_approval == true" in b
      effective = [write_owner] if guarded else principals
      for p in effective:
          agents[p]["allow_tools"].add(action)
  payload = {
    "generated_from": "tools-generated.cedar",
    "generator_version": "f1-v0.1",
    "generated_at": datetime.datetime.utcnow().isoformat() + "Z",
    "agents": { a: {
        "allow_tools": sorted(v["allow_tools"]),
        "deny_tools":  sorted(v["deny_tools"]),
        "notes": "Generated. Write-mode tools rerouted to human-operator."
                  if a == write_owner else
                  "Generated. Read-only tools mirrored from Cedar permits."
    } for a, v in agents.items() }
  }
  with open(out, "w") as f:
    json.dump(payload, f, indent=2, sort_keys=True)
  print(f"[ok] wrote {out} (agents: {len(payload['agents'])})")
  PY

  jq -e . "$OUT" >/dev/null
  if command -v ajv >/dev/null; then
    ajv validate --spec=draft2020 -s "$SCHEMA" -d "$OUT"
  fi

## Appendix B — Rollback plan

If F1 needs to be reverted after merging:

1. `git revert <PR-B-merge-sha>` — restores recipes to their pre-frontmatter state. Generator output collapses to header-only; nothing else changes because the generator was already a no-op against undecorated recipes.
2. `git revert <PR-A-merge-sha>` — removes generator + CI + Pi adapter + schema + lefthook hook. `.gitignore` entry vanishes. The hand-written `tools.cedar` continues to function exactly as before F1.
3. Run `pnpm install` to refresh the lefthook config (the removed hook stops applying).

Because no F2/F3 code consumes `tools-generated.cedar` yet, F1 rollback is local to this phase. After F2 ships, the rollback also requires reverting any gateway change that hard-references the generated file.
